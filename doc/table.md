| 0) Security standards & governance (“guardrails first”)                     | AppSec leadership, Platform engineering, Risk/Compliance | Define secure coding standards, LLM usage policy, approved OSS policy, severity SLAs, “golden paths” templates, exception process, and reporting.                     | **Atlassian Intelligence / Rovo** for summarizing standards, decision logs, and policy updates inside Confluence/Jira. ([Atlassian Support][1])                                                                                                                                                                                                             |
| 1) Requirements & user stories (“secure requirements”)                      | Product owner, Business analyst, Security champion       | Turn security needs into backlog items: data classification, authn/z, audit logging, privacy, abuse cases, acceptance criteria, non-functional requirements.          | **Atlassian Intelligence / Rovo** to summarize epics, extract action items, and keep requirements consistent across teams. ([Atlassian Support][1])                                                                                                                                                                                                         |
| 2) Architecture & threat modeling (“design it secure”)                      | Solution architect, Tech lead, AppSec                    | Create data-flow diagrams, identify trust boundaries, threat model, pick controls (rate limits, secrets mgmt, encryption, WAF), and produce mitigations + test cases. | **IriusRisk “Jeff” AI assistant** to accelerate threat model creation from prompts/diagrams and speed practitioner workflow. ([iriusrisk.com][2])                                                                                                                                                                                                           |
| 3) Developer inner loop (IDE) (“catch issues while coding”)                 | Developers, Security champions                           | Secure-by-default code patterns, avoid vulnerable APIs, prevent secret leaks, generate safer code snippets, and get instant feedback.                                 | **GitHub Copilot + Copilot Autofix (GHAS)** for fix suggestions tied to CodeQL alerts. ([GitHub Docs][3])  • **Snyk Code + Snyk Agent Fix** for AI-assisted remediation and validated fixes in dev workflows. ([Snyk][4]) • **GitGuardian** for secrets detection + AI-assisted workflows (incl. MCP server for AI IDE/agents). ([docs.gitguardian.com][5]) |
| 4) Pull request & code review (“secure merges”)                             | Developers, Code owners, AppSec reviewers                | Enforce secure review checklist, explain findings, reduce false positives, apply safe fixes, block high-risk merges.                                                  | **Semgrep Assistant** for AI-powered triage/remediation guidance on findings. ([Semgrep][6]) • **Snyk Fix PRs / PR experience** to propose/enable remediation via pull requests. ([docs.snyk.io][7]) • **Copilot Autofix** for CodeQL alert fix suggestions in PR flow. ([GitHub Docs][3])                                                                  |
| 5) Build-time dependency & supply chain (“secure what you ship”)            | Build/CI owners, Dev leads, AppSec                       | Dependency scanning, license policy, SBOM, prevent vulnerable transitive deps, automate upgrade PRs, enforce “no critical vulns” gates.                               | **Snyk** (automated fix PRs + AI remediation where available) to keep dependency risk down continuously. ([docs.snyk.io][7]) *(If you also use GHAS: pair with Dependabot + CodeQL; Copilot Autofix helps on code alerts.)* ([GitHub Docs][3])                                                                                                              |
| 6) IaC & cloud config shift-left (“secure infra before deploy”)             | Cloud/DevOps, Platform engineering, CloudSec             | Scan Terraform/K8s manifests, enforce secure modules, detect risky exposure (public buckets, overly broad IAM), prevent misconfig merges.                             | **Wiz + Ask AI (Mika)** to query risk contextually and drive ownership/remediation across teams. ([wiz.io][8]) *(Also commonly paired with policy-as-code scanners in CI; Wiz is often the “context + prioritization” layer.)* ([wiz.io][9])                                                                                                                |
| 7) Security testing automation (“prove mitigations work”)                   | QA/SDET, AppSec, Dev leads                               | Generate security test cases from threats, fuzz critical parsers, expand unit/integration test coverage for authn/z, input validation, and error handling.            | **Semgrep Assistant** for remediation guidance + reducing time-to-fix on SAST findings. ([Semgrep][6]) • **Snyk Agent Fix** to speed fixing code issues found pre-merge/CI. ([docs.snyk.io][10])                                                                                                                                                            |
| 8) Pre-prod validation (DAST, API scanning) (“shift-left runtime behavior”) | AppSec, QA, Release engineering                          | Run DAST/API scans in staging, validate auth flows, detect injection/SSRF, verify headers/cookies/CSP, confirm rate limits.                                           | *(Tool choice varies a lot here.)* Use AI mainly to **triage + generate repro steps + propose fixes**, typically via **Semgrep Assistant** / **Snyk** workflows. ([Semgrep][6])                                                                                                                                                                             |
| 9) Release & change management (“safe rollout”)                             | Release managers, SRE, AppSec                            | Verify gates met, ensure exceptions documented, generate release risk summary, confirm rollback/runbooks.                                                             | **Atlassian Intelligence / Rovo** to summarize readiness across Jira tickets + change records and keep audit trails readable. ([Atlassian Support][1])                                                                                                                                                                                                      |
| 10) Production monitoring & incident response feedback loop (“learn fast”)  | SOC, SRE, AppSec                                         | Triage incidents, generate hunting queries, speed investigations, route findings back into backlog, update detections/playbooks.                                      | **Microsoft Security Copilot + Sentinel** to analyze incidents and generate hunting queries using Sentinel data. ([Microsoft Learn][11]) • **Wiz Ask AI** for cloud risk/attack-path context feeding SOC-to-Eng remediation. ([wiz.io][8])                                                                                                                  |
